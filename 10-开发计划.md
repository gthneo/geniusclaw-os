## 一、模块划分总览

根据班迪提出的7大功能模块，开发计划如下：

```
┌─────────────────────────────────────────────────────────────┐
│                    GeniusClaw OS 开发                       │
├─────────────┬─────────────┬─────────────┬─────────────────┤
│  核心层      │  服务层      │  应用层      │  基础设施        │
├─────────────┼─────────────┼─────────────┼─────────────────┤
│ M0.基础镜像  │ M3.Skills   │ M6.路由出口  │ Docker/Compose  │
│ M1.OpenClaw │ M4.大模型   │ M7.NAS存储   │ Ubuntu 22.04    │
│ M2.P2P网络   │ M5.Agent    │ M8.外设驱动  │ MongoDB/Redis   │
│             │   市场      │ M9.Web3对接  │                 │
└─────────────┴─────────────┴─────────────┴─────────────────┘
```

## 二、模块开发详细计划

### 📦 M0 - 基础系统镜像（Foundation）

**目标**：构建 Ubuntu 基础镜像，预装必要组件

| 阶段 | 任务 | 交付物 |
|------|------|--------|
| M0.1 | Ubuntu 22.04 裁剪优化 | minimal-ubuntu-base.img |
| M0.2 | Docker/Compose 安装 | docker-compose.yml |
| M0.3 | MongoDB/Redis 集成 | 数据库服务配置 |
| M0.4 | 基础网络配置 | 网络初始化脚本 |
| M0.5 | 镜像打包工具链 | build-scripts/ |

**预计工作量**：2周

---

### 🔥 M1 - OpenClaw 集成（Core）

**目标**：将 OpenClaw 深度集成到系统

| 阶段 | 任务 | 交付物 |
|------|------|--------|
| M1.1 | OpenClaw 核心安装 | openclwd 服务 |
| M1.2 | Channel 配置 | channels/*.json |
| M1.3 | Bot 集成 | feishu/telegram/discord bots |
| M1.4 | Skills 系统集成 | skills runtime |
| M1.5 | 本地模型支持 | MiniMax/Ollama 集成 |

**预计工作量**：3周

---

### 🔗 M2 - P2P 网络服务（Core Feature）

**目标**：构建去中心化 P2P 网络

| 阶段 | 任务 | 交付物 |
|------|------|--------|
| M2.1 | P2P 核心库集成 | libp2p/helia |
| M2.2 | Boot Server 部署 | boot-server 服务 |
| M2.3 | NAT 穿透实现 | ICE/STUN/TURN |
| M2.4 | 节点发现与连接 | peer-discovery |
| M2.5 | 远程维护授权 | auth-flow |
| M2.6 | 端到端加密 | wireguard/QUIC |

**预计工作量**：4周

---

### 🎯 M3 - Skills Hub（Services）

**目标**：Skills 应用的集散中心

| 阶段 | 任务 | 交付物 |
|------|------|--------|
| M3.1 | Skills Hub 服务端 | skills-hub-api |
| M3.2 | Skills 存储结构 | SQLite + File Storage |
| M3.3 | 上传/下载流程 | upload/download UI |
| M3.4 | 分类与检索 | search & filter |
| M3.5 | 版本管理 | version-control |
| M3.6 | 收益分成系统 | billing-module |

**预计工作量**：3周

---

### 🤖 M4 - 大模型供应链（Services）

**目标**：多模型热插拔管理

| 阶段 | 任务 | 交付物 |
|------|------|--------|
| M4.1 | Ollama 集成 | local-model-runtime |
| M4.2 | API 网关 | openai-compatible-api |
| M4.3 | 模型热插拔 | model-switcher |
| M4.4 | Token 管理 | usage-tracker |
| M4.5 | 模型市场接入 | model-marketplace-api |
| M4.6 | 开源模型管理 | llama.cpp/ollama models |

**预计工作量**：3周

---

### 🤖 M5 - Agent 智能体市场（Application）

**目标**：Agent 线上交易市场

| 阶段 | 任务 | 交付物 |
|------|------|--------|
| M5.1 | Agent 注册系统 | agent-registry |
| M5.2 | 采购商 API | buyer-agent-api |
| M5.3 | 交易引擎 | HTTP 402 支持 |
| M5.4 | 行业细分市场 | marketplace-categories |
| M5.5 | 支付网关（可选） | digital-cny-integration |

**预计工作量**：4周

---

### 🌐 M6 - 路由出口（Application）

**目标**：VPN 与隐私保护

| 阶段 | 任务 | 交付物 |

| M6.2 | 广告拦截 | DNS/Hosts blocker |

| M6.4 | 隐私保护 | anti-tracking |
