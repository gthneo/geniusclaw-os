## 1. P2P 网络概述

### 1.1 设计目标

| 目标 | 说明 |
|------|------|
| **去中心化** | 无单一故障点，节点平等通信 |
| **内网穿透** | 支持 NAT 后的节点直连 |
| **安全加密** | 端到端加密，保护隐私 |
| **带宽共享** | 节点间互助传输 |

### 1.2 网络架构

```
┌─────────────────────────────────────────────────────────┐
│                    P2P Overlay Network                  │
│                                                          │
│   ┌──────────┐      ┌──────────┐      ┌──────────┐    │
│   │  Seed 1  │◄────►│  Seed 2  │◄────►│  Node 1  │    │
│   │ (中继)   │      │  (中继)  │      │ (用户)   │    │
│   └──────────┘      └──────────┘      └──────────┘    │
│        │                  │                  │        │
│        └──────────────────┼──────────────────┘        │
│                            ▼                           │
│                    ┌──────────────┐                   │
│                    │   Full Mesh  │  ← 可选直连       │
│                    │  (节点发现)  │                   │
│                    └──────────────┘                   │
└─────────────────────────────────────────────────────────┘
```

## 2. 种子节点配置

### 2.1 官方种子节点

| 节点 | 地址 | 状态 |
|------|------|------|
| Seed 1 | `seed1.geniusclaw.com:8443` | ✅ 在线 |
| Seed 2 | `seed2.geniusclaw.net:8443` | ✅ 在线 |
| Seed 3 | `5.231.56.249:8443` | ✅ 在线 |

### 2.2 节点配置示例

```yaml
# /etc/geniusclaw/p2p.yaml
p2p:
  # 种子节点列表（启动时连接）
  bootstrap_nodes:
    - "/dns/seed1.geniusclaw.com/tcp/8443/p2p/QmSeed1..."
    - "/dns/seed2.geniusclaw.net/tcp/8443/p2p/QmSeed2..."
    - "/ip4/5.231.56.249/tcp/8443/p2p/QmSeed3..."
  
  # 监听配置
  listen:
    - "/ip4/0.0.0.0/tcp/8443"
    - "/ip4/0.0.0.0/udp/8443"
  
  # NAT 穿透
  nat:
    enabled: true
    hole_punching: true
    relay_enabled: true
```

## 3. 网络协议

### 3.1 协议栈

| 层级 | 协议 | 说明 |
|------|------|------|
| 传输层 | TCP/UDP | 基础传输 |
| 加密层 | TLS 1.3 | 传输加密 |
| 认证层 | Noise Protocol | 身份认证 |
| 应用层 | Custom / Libp2p | 消息封装 |

### 3.2 连接流程

```
客户端                          种子节点
   │                               │
   │──── DNS 发现 ────────────────►│
   │                               │
   │◄─── 返回节点列表 ─────────────│
   │                               │
   │──── 建立连接 ────────────────►│
   │      (Noise 握手)             │
   │                               │
   │◄─── 连接确认 + 节点信息 ──────│
   │                               │
   │──── 广播自身存在 ────────────►│
   │      (DHT 更新)              │
   │                               │
   │◄─── 发现其他对等节点 ─────────│
```

## 4. NAT 穿透方案

### 4.1 穿透技术

| 技术 | 适用场景 | 成功率 |
|------|----------|--------|
| ** Hole Punching** | 对称 NAT | 60-70% |
| **TURN 中继** | 严格 NAT | 99% |
| **UPnP** | 家用路由 | 50% |
| **端口映射** | 手动配置 | 100% |

### 4.2 中继服务器

```yaml
relay:
  enabled: true
  
  # 中继节点（需要固定公网IP）
  relay_servers:
    - "relay1.geniusclaw.com:8443"
    - "relay2.geniusclaw.net:8443"
  
  # 何时使用中继
  fallback:
    - "直连失败时"
    - "对称型 NAT"
    - "企业防火墙内"
```

## 5. VPN 路由功能

### 5.1 VPN 架构

```
┌─────────────────────────────────────────────────────────┐
│                    GC OS 路由器                          │
│                                                          │
│  ┌─────────┐    ┌─────────┐    ┌─────────┐              │
│  │ 内网 LAN │◄──►│ VPN 网关 │◄──►│ P2P 网络 │            │
│  │ 192.168.x.x  │          │    │         │            │
│  └─────────┘    └─────────┘    └─────────┘              │
│                        │                                │
│                        ▼                                │
│                 ┌──────────────┐                        │
│                 │  出口节点    │  → 互联网              │
│                 └──────────────┘                        │
└─────────────────────────────────────────────────────────┘
```

### 5.2 路由模式

| 模式 | 说明 | 用途 |
|------|------|------|
| **全路由** | 所有流量走 VPN | 隐私保护 |
| **分路由** | 只转发指定流量 | 节省带宽 |
| **网关模式** | 作为路由器使用 | 家庭网络 |

### 5.3 WireGuard 配置

```yaml
wireguard:
  # 接口配置
  interface:
    private_key: "YOUR_PRIVATE_KEY"
    address: "10.0.0.2/24"
    dns:
      - "1.1.1.1"
      - "8.8.8.8"
  
  # 对等节点（出口节点）
  peer:
    public_key: "出口节点公钥"
    endpoint: "wg.geniusclaw.com:51820"
    allowed_ips: "0.0.0.0/0"  # 全路由
    persistent_keepalive: 25
```

## 6. 广告拦截

### 6.1 拦截原理

```
用户请求 ──► GC OS 路由 ──► DNS 过滤 ──► 允许/拦截
                      │
                      ▼
              ┌─────────────┐
              │ 广告黑名单  │
              │ (50000+域) │
              └─────────────┘
```

### 6.2 配置示例

```yaml
ad_block:
  enabled: true
  
  # 广告源
  sources:
    - "https://raw.githubusercontent.com/StevenBlack/hosts/master/hosts"
    - "https://adaway.org/hosts.txt"
  
  # 本地过滤规则
  local_rules:
    - "*.advertisement.local"
    - "tracker.*.com"
  
  # 白名单（不解拦截）
  whitelist:
    - "ads.example.com"  # 误报可在此添加
```

