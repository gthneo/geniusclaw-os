## 二、核心模块说明

### 2.1 主入口 (index.js)

**文件**：`src/index.js`

**类**：`GeniusClawOS`

**功能**：操作系统主入口，协调所有服务启动和停止

**主要方法**：

| 方法 | 说明 |
|------|------|
| `constructor(config)` | 初始化配置 |
| `init()` | 初始化所有服务 |
| `start()` | 启动操作系统 |
| `stop()` | 停止操作系统 |
| `getStatus()` | 获取系统状态 |

**配置项**：

```javascript
{
  p2p: {
    port: 4000,              // P2P 网络端口
    bootNodes: [],          // 引导节点列表
    enableRemoteMaintenance: false
  },
  openclaw: {
    host: 'localhost',
    port: 8080
  },
  mongodb: { uri: '...' },
  redis: { url: '...' },
  api: { port: 3000 }
}
```

**使用示例**：

```javascript
import { GeniusClawOS } from './index.js';

const os = new GeniusClawOS({
  p2p: { port: 4000 },
  api: { port: 3000 }
});

await os.init();
await os.start();

// 获取状态
const status = os.getStatus();
console.log(status);
```

### 2.2 P2P 网络服务 (p2p-network.js)

**文件**：`src/core/p2p-network.js`

**类**：`P2PNetworkService`

**功能**：P2P 网络核心，提供节点发现、连接、远程维护、市场交易等功能

**继承**：`EventEmitter`

**主要属性**：

| 属性 | 类型 | 说明 |
|------|------|------|
| `peerId` | string | 本节点 ID |
| `connectedPeers` | Map | 已连接节点 |
| `authorizedPeers` | Set | 授权远程维护的节点 |

**主要方法**：

| 方法 | 说明 |
|------|------|
| `start()` | 启动 P2P 网络 |
| `stop()` | 停止 P2P 网络 |
| `connectToBootNodes()` | 连接引导节点 |
| `authorizePeer(peerId)` | 授权远程维护 |
| `revokeAuthorization(peerId)` | 撤销授权 |
| `requestRemoteMaintenance(peerId, command)` | 请求远程维护 |
| `getConnectedPeers()` | 获取已连接节点列表 |
| `publish(topic, message)` | 发布消息到主题 |
| `subscribe(topic, handler)` | 订阅主题 |
| `getMultiaddrs()` | 获取本节点多地址 |

**消息类型**：

```javascript
{
  HELLO: 'hello',                      // 节点问候
  MAINTENANCE_REQUEST: 'maintenance_request',  // 远程维护请求
  MAINTENANCE_RESPONSE: 'maintenance_response', // 远程维护响应
  DATA_SYNC: 'data_sync',              // 数据同步
  MARKETPLACE: 'marketplace'           // 市场交易
}
```

**协议处理**：

- `/geniusclaw/maintenance/1.0.0` - 远程维护协议
- `/geniusclaw/data/1.0.0` - 数据同步协议
- `/geniusclaw/marketplace/1.0.0` - 市场交易协议

**使用示例**：

```javascript
import { P2PNetworkService } from './core/p2p-network.js';

const p2p = new P2PNetworkService({
  port: 4000,
  bootNodes: ['/ip4/1.2.3.4/tcp/4000']
});

await p2p.start();

// 监听节点事件
p2p.on('peer:connected', (peerId) => {
  console.log('New peer connected:', peerId);
});

// 授权远程维护
p2p.authorizePeer('Qmxxx...');

// 发布订阅
await p2p.subscribe('marketplace', (data, from) => {
  console.log('Received:', data);
});
```

## 三、P2P Agent 市场集成

### 3.1 Agent 标识结构

```javascript
{
  agentId: "gc-agent-0x7a2f...e3d1",     // 全网唯一 ID
  hostId: "gc-host-0x9b4c...f2e8",       // 主机 ID
  publicKey: "ed25519-...",              // 公钥
  blockchainAddress: "0x...",           // 区块链地址（可选）
  capabilities: ["skill:file-manager", "skill:ai-chat"], // 能力清单
  endpoint: "/p2p/QmX...",               // P2P 地址
  status: "online" | "busy" | "offline"
}
```

### 3.2 市场交易流程

```
发现 → 询价(HTTP 402) → 报价 → 支付 → 执行 → 确认
```
