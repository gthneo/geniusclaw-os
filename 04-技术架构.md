### 1.2 P2P 网络架构（核心特性）

```
                    ┌─────────────────┐
                    │   Boot Server   │
                    │   (引导节点)    │
                    │   首次启动注册  │
                    └────────┬────────┘
                             │
         ┌───────────────────┼───────────────────┐
         │                   │                   │
         ▼                   ▼                   ▼
   ┌───────────┐       ┌───────────┐       ┌───────────┐
   │  Host A   │◄─────►│  Host B   │◄─────►│  Host C   │
   │ (主节点)  │       │ (对等节点)│       │ (对等节点)│
   │           │◄─────►│           │◄─────►│           │
   └───────────┘       └───────────┘       └───────────┘
        │
   经用户授权
   可远程维护
        │
        └───────────────────────────────────────┘
                         │
                         ▼ (可选)
              ┌─────────────────────┐
              │  数字货币/代币特征  │
              │  (国外社区开发)      │
              └─────────────────────┘
```

### 1.3 P2P 连接特点

| 特性 | 描述 |
|------|------|
| **启动连接** | 首次启动连接 Boot Server，后续 P2P 直连 |
| **NAT穿透** | 支持 ICE/STUN/TURN 穿透各种网络环境 |
| **用户授权** | 需用户授权才能远程维护 |
| **端到端加密** | 所有 P2P 流量加密传输 |
| **分布式** | 无中心服务器，节点平等互联 |
| **数字货币特征** | 引导后可形成 P2P 市场，引入代币激励 |
