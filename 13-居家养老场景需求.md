## 一、需求来源

### 1.1 修伟分享（2026-03-17）

**产品定位**：Geniusclaw 真龙侠数字机器人（个人级/尖刀班）

## 三、应用场景

### 3.1 场景一：信息防骗

**场景**：老太太每天收到保健品推销电话和短信

**解决方案**：
- 真龙侠提供**客观真实的健康信息**
- 无推销、无广告诉求
- 帮助老人识别保健品营销话术

**对话示例**：
```
老人：有个保健品说能治糖尿病，靠谱吗？
真龙侠：保健品不能治病，只能辅助...
```

### 3.2 场景二：陪伴聊天

**场景**：老人子女不在身边，缺乏交流

**解决方案**：
- 真龙侠 24 小时在线聊天
- 聊新闻、天气、生活琐事
- 主动关心、情感陪伴

**对话示例**：
```
老人：今天天气怎么样？
真龙侠：今天晴天，15-22度，适合外出...
老人：我有点无聊
真龙侠：给您讲讲今天的新闻吧？...
```

### 3.3 场景三：生活助手

**场景**：老人不会用手机挂号、查天气、设提醒

**解决方案**：
- 语音/文字指令完成操作
- 提醒服药、提醒复查
- 简化生活操作

**对话示例**：
```
老人：帮我设置明天上午9点吃药的提醒
真龙侠：已设置，明天上午9点提醒您服药
```

### 3.4 场景四：家庭联络

**场景**：老人想联系子女但不会打视频

**解决方案**：
- 一键呼叫子女
- 语音指令发送消息
- 紧急联系人功能

## 四、产品需求

### 4.1 核心需求（Must Have）

| # | 需求 | 说明 |
|---|------|------|
| **R1** | **一键启动** | 像微波炉一样，插电就用 |
| **R2** | 简单交互 | 语音/文字对话，无需学习 |
| **R3** | 信息可信 | 提供客观、无推销的健康信息 |
| **R4** | 陪伴聊天 | 24小时在线，情感支持 |
| **R5** | 本地部署 | 数据隐私，不上传云端 |

### 4.2 重要需求（Should Have）

| # | 需求 | 说明 |
|---|------|------|
| R6 | 提醒功能 | 服药、复查、子女来电 |
| R7 | 紧急呼叫 | 一键呼叫子女/急救 |
| R8 | 简单设置 | 大字体、大音量、方言支持 |

| # | 需求 | 说明 |
|---|------|------|
